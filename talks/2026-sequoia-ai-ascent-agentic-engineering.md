## 核心观点

### 从 Vibe Coding 到 Agentic Engineering

Karpathy 回顾了一年前提出的 **"Vibe Coding"** 概念，指出：

- Vibe Coding 是让 AI 帮你写代码，你只需要"感受着走"
- 但更严肃的范式正在形成——**Agentic Engineering**（代理式工程），这是在 Vibe Coding 之上的更严谨的工程实践
- 两者不是替代关系，而是层次关系：Vibe Coding 是浅层，Agentic Engineering 是深层

### 三个关键比喻

1. **LLM 不是动物，而是"鬼魂"**（ghosts）
   - 说动物意味着它有某种生物的一致性
   - 鬼魂更贴切——模糊的、统计性的、被召唤出来的实体
   - 需要一种全新的品味和判断力来引导它们

2. **Agent 是安装器**（installer）
   - Software 3.0 需要一个新的安装层
   - 就像软件需要通过安装器来分发，AI Agent 就是 Software 3.0 的安装器

3. **"菜单模式 vs 原始提示"**
   - 用餐厅做比喻：菜单（结构化界面）适合确定性任务
   - 原始提示框（raw prompts）适合探索性任务
   - 两者各有适用场景

### Software 3.0 的演进

- Software 1.0：手写代码
- Software 2.0：神经网络（训练模型）
- Software 3.0：AI 替你写代码
- 关键洞察：Software 3.0 的核心不是写代码，而是**理解**——你可以外包你的思考（让 AI 写代码），但永远不能外包你的理解

### 可验证性与锯齿形技能（Jagged Skills）

- AI 的能力分布不是平滑的，而是**锯齿形**的——有些任务做得极好，有些却出人意料地差
- 可验证性成为关键：当 AI 产出无法被验证时，它就不可靠
- 在可验证的领域（如编程），AI 的效果最好
- 在开放式的、无法验证的领域，人类判断仍然不可替代

### 关于"越来越感觉自己落后"

- Karpathy 坦言作为程序员从未感觉如此落后
- 不是因为 AI 比我强，而是因为**知识增长曲线变陡了**
- 需要学习的东西爆炸性增长，新技术每天都在涌现
- 能给你最大回报的是：**学习品味**（taste）和**判断力**（judgment）

## 给创始人的建议

1. **自动化一切可以自动化的**——不要自满，AI 的能力扩张速度超过大多数人的预期
2. **不要外包你的理解**——即使 AI 帮你写代码，你仍然需要理解系统在做什么
3. **培养品味**——知道什么是好的、什么是重要的，这比任何技术技能都重要
4. **关注可验证性**——在你能验证结果的领域用 AI，在不能验证的领域保留人类判断
